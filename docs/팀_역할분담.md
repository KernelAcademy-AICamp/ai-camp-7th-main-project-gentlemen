# 팀 역할 분담

> 4개 work-stream. 한 명(리드)이 통합+생성을 들고, 나머지가 품질·백엔드·프론트를 나눠 맡는다.
> 영역 경계·규칙: [CONTRIBUTING.md](../CONTRIBUTING.md) · 전체 작업: [작업트랙.md](작업트랙.md)

## 역할 → 디렉토리 → 첫 태스크

### A. 카드뉴스 품질 (Content/Generation)
- **디렉토리**: `lib/generate/` · `lib/llm/` · `lib/render/` · `poc/`
- **하는 일**: 프롬프트 고도화·모범예시(골든셋)·템플릿 디자인·수동 카드 제작·LLM 라우팅(벤치)
- **첫 태스크**:
  1. `lib/llm/`에 **실제 Anthropic 어댑터** 추가 (mock→실모델) — 키는 `poc/.env`
  2. 3시나리오(재테크/디저트 고위험/자유주제) 실제 생성·렌더 → 품질 눈으로 점검
  3. 글자수 제약 vs 내용 깊이 튜닝, 자가점검(ai_flags) 정확도 올리기
- 근거: [생성 파이프라인](tech/Kup_생성파이프라인_설계.md) · [프롬프트 설계](tech/Kup_프롬프트_설계.md)

### B. 백엔드 / 인프라 (Backend)
- **디렉토리**: `workers/` · `supabase/` · `lib/db/` · `lib/supabase/`
- **하는 일**: 예약 발행·인사이트 cron·DB/마이그레이션·IG OAuth/토큰·Redis·Sentry
- **첫 태스크**:
  1. `spike-bullmq`의 cron/delayed 로직을 **`workers/`(TS)로 이식** (Phase 5)
  2. IG OAuth 정식 플로우 + 토큰 암호화 저장(`ig_tokens`)
  3. 렌더 PNG → Supabase Storage → 공개 URL
- 근거: [데이터모델](tech/Kup_데이터모델.md) §7 · [BullMQ 심층](tech/Kup_BullMQ_cron_심층.md) · `spike-bullmq/README.md`

### C. 프론트엔드 / 제품 UI (Frontend)
- **디렉토리**: `app/`
- **하는 일**: 프로토타입 → Next.js: 워크스페이스 셸·칸반/캘린더·대시보드/인사이트·**검수 화면**·온보딩/연동 UI
- **첫 태스크**:
  1. 워크스페이스 셸 + 사이드바 내비 (프로토타입 기준)
  2. **검수 화면** — DB의 deck 로드 + `ai_flags`/`risk_level` 바인딩 + 고위험 하드게이트
  3. 대시보드/칸반 (deck status 6단계)
- 근거: [화면흐름](product/Kup_화면흐름.md) · 시각 SoT: `docs/product/kup-prototype-lowfi.html`

### D. 리드 / 통합 (Lead)
- **하는 일**: 아키텍처·공유 계약(deck-schema/DB타입)·코드리뷰·CI·배포(Vercel/Railway)·머지 케이던스
- 이미 보유: 생성 파이프라인·영속화 토대·스파이크 검증

## 협업 원칙
- 서로 **계약(인터페이스)에 맞춰 따로** 일한다 → 충돌 최소화.
  - 생성 결과 = `Deck`(deck-schema), 저장 단위 = `decks` 테이블, 화면은 그 deck을 읽어 그림.
- 공유 계약 변경은 **사전 공유 + PR 리뷰**([CONTRIBUTING §4](../CONTRIBUTING.md)).
- 막힌 외부 의존: API 키(품질은 `poc/.env`로 해소됨) · 호스팅 계정(배포 시) · IG 앱 리뷰(공개 출시 전).
