## 🌿 git worktree 운영 원칙 (기술 트랙)

- **브랜치 성격 구분**: 스파이크(`spike/*`, 버려도 됨) vs 제품 뼈대(`feat/*` 또는 `dev`, **merge 대상**). Task 3+는 후자.
- **worktree 개수 최소화**: main + dev 1개. task마다 새로 파지 않는다.
- **worktree마다 독립**: `node_modules`·`.env`는 공유 안 됨 → 각 worktree에서 `npm install`+자체 `.env`. 시크릿은 절대 커밋 금지.
- **merge 케이던스**: 안정 마일스톤(뼈대 빌드 성공, 각 Phase 완료)마다 dev→main 머지 → main에서 기획 문서와 코드가 수렴.
- **머신 간 동기화**: worktree는 머신 로컬. 노트북 2대는 **브랜치 push/pull로 동기화**, worktree는 머신별로 각자 생성.
