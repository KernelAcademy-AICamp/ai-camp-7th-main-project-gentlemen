# Kup — 인스타 연동 따라하기 (팀 공유용)

> 목적: 새 인스타 계정을 우리 서비스(테스트 앱)에 연결해서, **우리 코드로 발행/연동이 되는지 확인**하기.
> 작성 기준: 2026년 6월. 방식 = **Instagram API with Instagram Login**.
> 진행 방식: 막히면 캡처해서 공유. 토큰 받는 단계는 혼자 하지 말고 **같이** 진행.

---

## 0. 큰 그림 — 누가 뭘 하나 (중요)

우리는 **메타 앱을 1개만** 씁니다. (이미 팀장이 만들어 둠 = "1kup test")
- **앱 주인(팀장) 1명**: 앱을 만들고, 권한을 켜고, 팀원 계정을 "테스터"로 등록. → **A파트** (이미 완료)
- **팀원들**: 페이스북·개발자등록·앱 만들기 **안 함.** 자기 인스타 계정을 그 앱에 **연결만** 함. → **B파트**

> 비유: 관리자가 **매장(앱)** 을 차렸고, 팀원은 그 매장에 **출입증(테스터 등록 + 토큰)** 만 받으면 됨.
> 이 구조가 나중에 만들 진짜 서비스(앱 1개에 사용자 여러 명 연결)와 똑같음.

**준비물 (팀원 공통)**
- [ㅇ] 새 인스타 계정 (지인 노출/알고리즘 섞임 방지 위해 새 이메일로 생성)
- [ㅇ] **프로페셔널(비즈니스/크리에이터) 계정으로 전환** ← 개인 계정은 API 불가
- [ ] 자기 인스타 **사용자명**을 관리자에게 알려주기

> 프로페셔널 전환: 인스타 앱 → 프로필 → ≡ 메뉴 → 설정 및 개인정보 → 계정 유형 및 도구 → 프로페셔널 계정으로 전환 → 카테고리 선택 → 비즈니스/크리에이터 선택. (무료·즉시·되돌리기 가능)

# A파트 — 관리자(앱 주인) 1회 설정  *(이미 완료된 기록 / 참고용)*

> 팀원은 이 파트를 **할 필요 없음.** 앱이 이미 있음. 어떻게 만들어졌는지 기록용.

### A-1. 페이스북 계정 만들기
- Meta 개발자 포털은 **페이스북 로그인만** 됨 → 새 이메일로 페이스북 계정 생성.
- ⚠️ 페이스북은 "개발자 포털 출입증"으로만 필요. 우리 서비스 사용자는 페이스북 필요 없음.

### A-2. 메타 개발자 등록
- **developers.facebook.com** 접속 → 페이스북 로그인 → 개발자 등록(전화/이메일 인증).
- "회원님을 가장 잘 설명하는 역할" 질문 → **개발자** 선택 (기능 영향 없음).

### A-3. 앱 만들기
- 내 앱 → **앱 만들기**
- 앱 유형: **비즈니스(Business)**
- 앱 이름: 예) `1kup test`

### A-4. 이용 사례 선택
- **"Instagram에서 메시지 및 콘텐츠 관리"** 하나만 선택. (발행+댓글+DM이 다 들어있음)
- 나머지는 선택 안 함. 나중에 추가·수정 가능.

### A-5. 비즈니스 포트폴리오
- **"아직 비즈니스 포트폴리오를 연결하고 싶지 않음"** 선택.
- (사업자 인증은 출시 직전 = 앱 검수 단계에서. 개발 모드 테스트엔 불필요.)

### A-6. 권한 추가 (이용 사례 맞춤 설정 → 권한 및 기능)
**"business" 붙은 것만** 켜면 됨. (`business` 없는 건 구버전용 → 무시)
- `instagram_business_basic` — 계정 기본정보 읽기
- `instagram_business_content_publish` — **발행(카드뉴스)** ← 꼭 추가
- `instagram_business_manage_comments` — 댓글 읽기(→DM)
- `instagram_business_manage_messages` — DM 발송
- (선택) `instagram_business_manage_insights` — 성과 지표(도달·팔로워)
- 각 항목 우측 **"+ 추가"** → 상태가 **"테스트 준비 완료"** 로 바뀌면 OK.

### A-7. 앱 ID / 시크릿 확보
- 좌측 **"Instagram 로그인이 포함된 API 설정"** 화면에서 **앱 ID**, **앱 시크릿** 복사.
- 이 둘은 `poc/.env` 의 `IG_APP_ID`, `IG_APP_SECRET` 에 저장. (시크릿 = 비밀, 공유 금지)

# B파트 — 팀원 온보딩 (각 팀원이 할 일)

> 핵심: 관리자가 너를 **테스터로 등록** → 너가 **수락** → 같이 **토큰 발급**. 끝.

### B-1. (팀원) 인스타 사용자명을 관리자에게 전달
- 프로페셔널 전환까지 마친 새 계정의 **@사용자명**을 관리자에게 알려줌.

### B-2. (관리자) 테스터로 등록
- 앱 대시보드 → **앱 역할(Roles)** → **사람 추가** → **Instagram 테스터** → 팀원 사용자명 입력 → 초대.
- 등록되면 그 계정이 **"대기 중"** 상태로 뜸.

### B-3. (팀원) 테스터 초대 수락
> ⚠️ 초대는 인스타 **알림(종 모양)에 안 뜸.** 설정 안쪽에서 직접 수락해야 함.

**방법 A — 인스타 앱**
1. 프로필 → ≡ 메뉴 → **설정 및 개인정보**
2. 상단 **검색창**에 `앱` 또는 `웹사이트` → **"앱 및 웹사이트"** 열기
3. **"테스터 초대"** 탭 → 우리 앱 → **수락**

**방법 B — 웹브라우저 (더 확실할 때 많음)**
1. instagram.com 에 **그 계정으로** 로그인
2. 주소창에 `https://www.instagram.com/accounts/manage_access/`
3. **테스터 초대** → **수락**

→ 수락하면 관리자 화면(역할 탭)에서 상태가 **"대기 중" → 수락됨**으로 바뀜(새로고침).

### B-4. (같이) 액세스 토큰 발급
> 토큰 = 우리 코드가 그 계정에 발행할 수 있게 해주는 "열쇠". 화면 흐름이 여러 개라 **관리자와 같이/화면공유로** 진행.

1. 앱 대시보드 → Instagram 설정 → **"2. 액세스 토큰 생성"** → **계정 추가**
2. 뜨는 팝업에서 **그 팀원의 인스타 계정으로 로그인** → 권한 허용
3. **액세스 토큰** 생성됨 → 복사 (⚠️ 비밀. 공개 채널 금지)

### B-5. (관리자/개발) 토큰을 코드에 넣고 연결 확인
- `poc/.env` 에 팀원 토큰을 넣고(팀원별로 관리) 연결 확인 실행:
  ```
  cd poc
  node --env-file=.env ig-test.js verify
  ```
- 성공하면 이렇게 나옴:
  ```
  ✅ 연결 성공! 계정 정보:
     사용자명 : (팀원 계정)
     계정유형 : BUSINESS (발행 가능)
     user_id  : 17841xxxxxxxxxxx   ← 이 계정의 고유 번호
  ```
- 이 `user_id` 를 `.env` 의 `IG_USER_ID` 에 넣어두면 됨. (안 넣어도 코드가 자동으로 알아냄)

> **user_id 란?** 인스타 계정마다 붙은 고유 번호(이름은 바뀌어도 이 번호는 그 계정을 가리킴). 보통 `17`로 시작하는 긴 숫자.

## C. 발행 테스트 (연결 성공 후)

> 인스타 발행은 2단계: **① 컨테이너 생성(준비)** → **② 게시(공개)**.
> 계정을 진짜 키울 거면, 먼저 **올리기 직전(①)까지만** 테스트해서 계정을 깨끗하게 유지하는 걸 권장.

전제: 카드 이미지는 **공개 URL** 이 있어야 인스타가 가져감 (= 이미지 호스팅 필요).
`.env` 의 `IMAGE_URL`(1장) / `IMAGE_URLS`(콤마로 2장 이상)에 공개 이미지 주소를 넣는다.

```
# [안전] 실제로 안 올라감 — 파이프라인만 검증
node --env-file=.env ig-test.js stage-image
node --env-file=.env ig-test.js stage-carousel

# ⚠️ 실제 공개 게시 (계정에 진짜 올라감)
node --env-file=.env ig-test.js post-image
node --env-file=.env ig-test.js post-carousel
```

> 알고리즘 걱정: 팔로워 0인 새 계정은 테스트 글 1~2개 올렸다 지운다고 의미 있게 망가지지 않음.
> 다만 진짜 키울 계정이면, 첫 게시물은 **버리는 테스트 말고 제대로 만든 콘텐츠**로 올리는 게 깔끔함.

## 막히면?
각 단계에서 **화면 캡처 / 보이는 텍스트 / 에러 메시지**를 공유. 토큰·시크릿은 일부 가려서.
토큰·시크릿은 **절대 공개 저장소(깃 등)에 올리지 말 것.** `poc/.env` 에만 두고, 이 파일은 `.gitignore`로 보호됨.
