## 부록 A. 킥오프 회의 원본 (보존용)

> 7개 직군(대표/PO/PM/PL/개발자/기획자/디자이너) 가상 회의. 현행 기획 4종을 자료로 검토.

### 라운드 1 — 각 직군 진단
- **대표**: 방향 선명. 우려 = ① 언제 과금(베타 무료+무료회원 DM 100건) ② '1,000명'이 보장처럼 들림(법무 검토 필요).
- **PO**: 1K는 후행지표라 베타에선 측정 불가 → 베타 성공지표를 **주간 발행 지속률 + 무수정 승인율** 2개로 압축 주장.
- **PM**: IA 범위가 MVP가 아님. 베타 무료면 **결제 모듈 통째로 범위 밖**.
- **PL**: 최대 리스크는 기술 아닌 **Meta 앱 검수**. → (※로미 정정: 베타 테스터 한정이라 **앱 검수 불필요**, Live 전환됨. 해소)
- **개발자**: 발행 API는 **프로 계정 전제** 필요 → (※로미 정정: 타깃이 **이미 프로 계정 보유자**. 모순 해소)
- **기획자**: DM 리드마그넷은 댓글 필요 → 0~100 신규계정은 댓글 없음(닭-달걀). 0~100 그로스 레버 부재.
- **디자이너**: AI 카드뉴스가 캔바보다 예쁜가? 양산형은 인스타가 도달 누름. 검수 UX 무거우면 '시간 단축'과 충돌.

### 라운드 2 — 쟁점
1. **타깃↔계정 모순** → ✅ 해소(타깃=프로계정 보유). 온보딩에 연동 가이드+안전 문구.
2. **0~100 그로스 공백** → 이 구간 주인공은 **발행 습관(챌린지)**, 리드마그넷은 100+부터.
3. **MVP 스코프 컷** → 결제/구독/증빙 v1.1로, 칸반 6→3열, 인사이트 9→3종 (※로미: **미적용**, 참고만).
4. **검수↔속도** → '플래그만 보는 5초 결정' → ✅ 로미가 **발행 내장 자동 검수**로 확정.

### 결정/정정 (로미 확정)
- 타깃 = 프로페셔널 계정 보유 입문자 (PRD/SPEC 반영 완료)
- 계정 연동에 가이드+안전 문구 (SPEC 반영 완료, IA 기존 보유)
- 앱 검수 불필요 · Live 전환 · 테스터 역할 등록 (메모리 기록)
- 검수 = 발행 버튼 내장 자동 게이트 (SPEC §3.3 반영 완료)
