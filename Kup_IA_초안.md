# Kup — IA (정보구조) 초안

> **문서 성격**: `kup_PRD_v1.0_최종본`을 화면·내비게이션 구조로 번역한 **IA 근거·스펙 문서**. 팀 IA 스프레드시트(`KUP IA`)가 **구조의 단일 진실(source of truth)**이고, 이 문서는 그 구조의 *설계 근거·화면 스펙·정합성 점검* 동반본. **설명(M열) 붙여넣기 초안은 `Kup_IA_설명_초안.md`** 참조.
> **기준**: 최종 PRD v1.0 · 팀 IA 스프레드시트(워크스페이스 0.2 / 마이페이지 0.3 / 회원가입 0.4 / 결제 0.5) · 매니페스트 기능명세서.
> **용어**: 콘텐츠 단위 = **프로젝트**(팀 IA 용어로 통일). 카드뉴스 1건 = 1프로젝트.
> **타깃 범위**: **단일 타깃 — "팔로워 0명에서 시작하는, 갓 시작한 1인 인플루언서"**. (소상공인은 이 안에 포함될 수 있으나 대상층을 둘로 나누지 않는다 → 소상공인 전용 모듈·분기는 제거.)
> **버전**: v1.0-draft · 작성 2026-06 (PRD v1.0 정합화)
> **읽는 순서**: ① 설계 원칙 → ② 사이트맵 → ③ 내비게이션 모델 → ④ 화면 인벤토리 → ⑤ 화면별 정보구조 → ⑥ 객체↔화면 매핑 → ⑦ 매니페스트 기능명세 ↔ IA 매핑 → ⑧ 확장 자리(P1/P2) → ⑨ Open Questions
> **depth 계약(IA 4레벨)**: ① 도메인(사이드바 섹션/큰 단계) → ② 화면(페이지, 와이어프레임 1장) → ③ 화면 내 섹션·블록 → ④ 행동·상태·예외. *매니페스트 트리는 "요구사항 루트 1개"라 도메인 레벨이 없음 → 이 IA가 도메인 분할을 담당한다.*

## ① IA 설계 원칙 (PRD 제품 원칙에서 도출)

| 원칙 | IA에 주는 규칙 |
|---|---|
| **주체성 보존** | 모든 콘텐츠의 종착점은 사용자의 **최종 컨펌**. 발행(Graph API)도 컨펌 후에만 일어나는 '승인 발행'이지 무검수 자동발행이 아니다. AI 산출물은 항상 *수정 가능한 초안* 상태로 노출. |
| **검수 = 직접 수정 + 최종 컨펌** | 초안 → **에디터** → **검수 게이트 + 최종 컨펌**을 끊기지 않는 한 흐름으로. 컨펌 화면 없이 발행·내보내기로 못 건너뜀(하드 게이트). |
| **발행/내보내기 병행** | 컨펌 후 두 경로 제공: **IG 발행(Graph API)** 또는 **복사·내보내기(직접 올림)**. 둘 다 'AI 생성 콘텐츠' 라벨·검수 통과 전제. |
| **시간 단축이 가치** | 홈은 '설정 모음'이 아니라 **"이번 주 할 일 + 진행 중 콘텐츠"** 허브. 제작까지 최소 클릭. |
| **성과→다음 액션 루프** | 성과는 숫자 나열이 아니라 **"다음에 바꿀 점"**으로 끝나고, 그게 다시 제작(주제 추천)으로 연결되는 순환. |
| **작은 목표** | 1,000을 단일 게이지로 두지 않고 **100/500/1,000 마일스톤 트래커**를 홈에 상시 노출. ('1,000 보장' 오해 방지 — PRD 리스크 4.) |
| **그로스 연결** | 발행한 게시물에서 **DM 리드마그넷(댓글 트리거 → 옵트인 DM)**으로 잠재 팔로워를 모으는 경로를 코어로 둔다. 스팸·정책 위반 방지 가드 포함(PRD 리스크 5.). |

> **단일 타깃 결정 메모**: 이전 IA 초안(v0.1)은 초보(A)+소상공인(C) 이중 세그먼트였으나, **최종 PRD v1.0은 타깃을 "갓 시작한 1인 인플루언서" 하나로 확정**. 따라서 소상공인 전용 화면(구 K5 비즈니스 문구 도구)·온보딩 분기·성과 분기를 **제거**한다. 소상공인 성향 사용자는 동일 코어 파이프라인을 사용한다.

## ② 전체 구조 (사이트맵)

**모델: 미리캔버스형 — 공개 인트로(소개) → [시작하기] → 로그인/회원가입 모달 → 워크스페이스(로그인 후 모든 작업의 단일 컨테이너).** 인스타 연동·콘텐츠 생성·발행·관리까지 전부 워크스페이스 안에서 일어난다.

```
Kup
│
├─ [공개 · 비로그인] 인트로
│   └─ L1 인트로 랜딩 (서비스 소개 + 주요 기능 미리보기 + [시작하기])
│        └─ L2 로그인 / 회원가입 모달(팝업)  ← [시작하기] 클릭 시 (이메일 · 구글)
│             └─ 인증 완료 → 워크스페이스 진입
│
└─ [워크스페이스]  ★로그인 후 모든 작업의 단일 셸 (좌측 사이드바 + 작업 영역)
    │
    ├─ (최초 1회 · first-run 가이드)  ← 워크스페이스 안에서 진행, 별도 풀스크린 라우트 아님
    │   ├─ O1 계정 목적·주제 입력
    │   ├─ O2 계정 상태 입력 (팔로워·기간·업로드 빈도·형식)
    │   ├─ O3 톤앤매너 설정
    │   ├─ O4 진단 결과 (운영 단계 + 이번 달/주 목표 + 추천 주제)
    │   ├─ O5 운영 체크리스트 (계정 기초 세팅)
    │   └─ O6 인스타그램 계정 연동                         ★코어
    │
    ├─ [홈] 홈 · 대시보드  ★허브
    │   └─ H1 오늘의 코치 (이번 주 목표·진행 중 콘텐츠·성과 입력 대기·마일스톤·다음 액션)
    │
    ├─ [콘텐츠] 제작 파이프라인
    │   ├─ K1 주제 추천 (이번 주 5개)
    │   ├─ K2 초안 생성 결과 (제작 패키지)
    │   ├─ K3 직접 수정 에디터
    │   ├─ K4 검수 게이트 & 최종 컨펌                       (하드 게이트)
    │   └─ K5 발행 & 내보내기 (IG 발행 Graph API / 복사·내보내기)  ★발행 코어
    │
    ├─ [성과·성장]
    │   ├─ P1 성과 기록 입력
    │   ├─ P2 다음 액션 추천
    │   ├─ P3 성과 히스토리 · 마일스톤
    │   └─ G1 DM 리드마그넷 (댓글 트리거 → 옵트인 DM)        ★코어
    │
    └─ [내 계정]
        ├─ A1 계정 홈 (현재 단계·프로필 요약)
        ├─ A2 톤앤매너 프로필 (편집)
        ├─ A3 운영 체크리스트 (재방문)
        ├─ A4 인스타그램 연동 관리 (재연동·권한·해제·계정 전환)  ★코어
        ├─ A5 승인 로그 (발행 전 컨펌 이력 조회)             ★코어
        └─ A6 설정 (구독·알림·계정)
```

★ = 워크스페이스 셸 / 발행·연동 등 코어 화면

**핵심 원칙**:
- **인트로(L1)** = 로그인 전 공개 마케팅 페이지. "이런 서비스예요" 소개 + 끝에 **[시작하기]** 하나. (네가 말한 '소개용 온보딩'이 이것 — 전략 진단 설문 O1~O6과는 다름.)
- **인증은 모달(L2)** — 별도 페이지가 아니라 [시작하기] 시 팝업. 로그인/회원가입(이메일·구글) 완료 즉시 **워크스페이스로 이동**.
- **워크스페이스 = 단일 작업 셸.** 온보딩 설문·IG 연동·제작·발행·성과·관리 **전부 여기 안**. 사이드바 4섹션이 워크스페이스의 뷰들.
- **온보딩 설문(O1~O6)은 워크스페이스 first-run.** 최초 진입 시 가이드 형태로 떠서 진단→연동→첫 제작으로 안내(풀스크린 위저드로 막아 세우지 않음). 완료 후엔 [내 계정]에서 값 수정.

## ③ 내비게이션 모델

**구조: 공개 인트로(L1) → [시작하기] → 로그인/회원가입 모달(L2) → 워크스페이스 셸(좌측 사이드바 4섹션 + 홈 허브).** **웹 우선**(데스크톱 웹 기준 설계, 모바일 웹은 반응형 — PRD 기기: 웹/모바일 웹)

```
데스크톱 웹 (워크스페이스 셸 · 기준)              모바일 웹 (반응형)
┌────────────────┬────────────────┐        ┌─────────────────────┐
│  [Kup 로고]     │                │        │ ☰ Kup          👤   │ ← 상단바
│ ┌────────────┐ │                │        ├─────────────────────┤
│ │＋인스타 계정 │ │                │        │                     │
│ │   연동하기   │ │   (작업 영역)   │        │     (작업 영역)       │
│ └────────────┘ │                │        │                     │
│  🏠 홈          │                │        │                     │
│  ✍️ 콘텐츠       │                │        ├─────────────────────┤
│  📊 성과·성장    │                │        │ 🏠   ✍️   📊   👤    │ ← 하단바(접힘)
│  👤 내 계정      │                │        └─────────────────────┘
│  [＋ 만들기]     │                │
└────────────────┴────────────────┘
```

- **워크스페이스 = 로그인 후 모든 작업의 단일 셸.** 좌측 사이드바 + 우측 작업 영역. 사이드바 4섹션이 워크스페이스의 뷰.
- **사이드바 최상단**: ① **서비스 로고**(Kup) → ② 그 아래 **아주 큰 [＋ 인스타 계정 연동하기] 버튼** (연동 전 항상 강조; **연동 완료 시 계정 칩/전환 UI로 대체**) → ③ 4섹션(홈/콘텐츠/성과·성장) → ④ 글로벌 **[＋ 만들기]**(어디서든 K1) → ⑤ 내 계정.
- **연동 게이트(소프트)**: IG **미연동 상태에서 제작·발행·성과 등 기능 클릭 시 → "SNS 계정 연동하기" 모달**(M1)을 띄움 — *"계정을 연동하면 콘텐츠 예약·관리·분석을 바로 시작할 수 있어요"* + [취소]/[계정 연동하기]. (둘러보기는 허용하되, 실제 작업 진입은 연동으로 유도. ⑤ M1 스펙 참고.)
- **비로그인(L1·L2)** 은 워크스페이스 밖(공개 마케팅 레이아웃). 인트로 → [시작하기] → 로그인/회원가입 모달 → 워크스페이스 진입. (최초 진입 시 온보딩 설문 O1~O6이 first-run 가이드로 등장.)
- **홈 섹션이 모든 행동의 시작점.** "이번 주 목표 → 추천 주제 → 진행 중 콘텐츠 → 성과 입력 대기"를 카드로 띄워 *다음에 뭘 할지 고민하지 않게* 한다. 웹 가로 폭을 살려 **멀티 칼럼 대시보드**로 배치.
- **콘텐츠 섹션은 제작 파이프라인.** K1→K2→K3→K4→K5가 한 방향 흐름. 중간 이탈 시 초안은 '진행 중'으로 저장돼 홈에 다시 뜬다. **K4(검수·컨펌) 통과 없이 K5(발행/내보내기)로 못 감.** (웹에선 에디터 K3를 좌(편집)/우(미리보기) 분할 가능.)
- **성과·성장 섹션은 루프의 후반 + 그로스.** 발행/내보낸 콘텐츠가 '성과 입력 대기'로 쌓이고(P1), 입력하면 다음 액션(P2)→다시 제작. 발행 게시물에 **DM 리드마그넷(G1)** 을 걸어 팔로워 유입.
- **내 계정 섹션은 무거운 설정 + 감사 이력.** 톤앤매너·체크리스트·IG 연동·승인 로그·구독.

**근거:** PRD 핵심 루프(진단→추천→초안→수정→검수→발행→성과→다음액션→그로스)는 *순환*이다. 홈 허브가 매 순간 "지금 할 한 가지"를 제시해 '꾸준히 못 함'(Pain #1)에 직접 대응한다. **웹 가로 공간**은 대시보드·에디터·잔디 그래프를 한눈에 펼치는 데 유리.

> **대안 검토:** ① 위저드만(탭 없이) → 반복 사용(주 2~3회 제작+성과)엔 섹션 재진입이 유리 → 사이드바 채택. ② 상단 탭바 vs 좌측 사이드바 → 섹션 4개+글로벌 만들기 버튼엔 **좌측 사이드바**가 확장(섹션 추가)에 유리. ③ DM 리드마그넷 별도 섹션 → MVP는 '성과·성장' 안에 둬 발행 직후 동선과 붙임(확장 시 섹션 승격 — ⑨ Open Q).

## ④ 화면 인벤토리

| ID | 화면 | 목적 | 핵심 콘텐츠 블록 | 주요 액션(CTA) | 진입 → 이탈 | 단계 |
|---|---|---|---|---|---|---|
| L1 | 인트로 랜딩 | 소개·시작 유도 | 서비스 소개·주요 기능 미리보기·신뢰 요소 | **[시작하기]** | 외부 유입 → L2(모달) | P0 |
| L2 | 로그인/회원가입 **모달** | 인증 | 이메일 가입·구글 연동·로그인 | [가입]/[로그인] | L1 [시작하기] → 워크스페이스 | P0 |
| M1 | SNS 계정 연동 유도 모달 | 미연동 시 연동 전환 | 안내 문구·아이콘 | [취소]/[계정 연동하기] | 미연동 상태 기능 클릭 → O6 | P0 |
| O1 | 목적·주제 입력 | 전략 진단 입력 | 운영 목적, 계정 주제 | [다음] | L2 최초 → O2 | P0 |
| O2 | 계정 상태 입력 | 진단 입력 | 팔로워·기간·주당 업로드·주 형식 | [다음] | O1 → O3 | P0 |
| O3 | 톤앤매너 설정 | 스타일 저장 | 선호 톤·피할 스타일·키워드·문체 예시·금지어·비주얼 | [다음] | O2 → O4 | P0 |

#### ⭐ 업로드 릴레이 (컨트리뷰션 그래프) — 표기 스펙

> **한 줄 정의**: GitHub 잔디(컨트리뷰션 그래프)를 인스타 발행에 적용한 **"꾸준함 시각화"**. 매일/매주 발행을 칸으로 쌓아 *끊기지 않게(릴레이)* 이어가도록 동기를 준다. PRD Pain #1(꾸준한 발행이 안 됨)과 핵심 지표(주간 발행 지속률·주간 게시 수)에 직접 대응.

**1. 1칸(셀)의 정의 — 무엇이 "잔디"인가**
- **1칸 = 하루.** 칸의 **농도 = 그날 발행 확정 건수**(0·1·2·3·4+ → 색 5단계). 빈 칸 = 그날 발행 0.
- **"발행 확정" 카운트 규칙**(잔디의 정직성이 핵심 — GitHub가 실제 커밋만 세듯):
  - **IG 발행(Graph API)** → 발행 성공 시 **자동 카운트** (timestamp 확정).
  - **복사·내보내기(직접 올림)** → 실제 업로드를 서버가 모름 → 사용자가 **"올렸어요" 확인** 시 카운트. (또는 IG 읽기 연동 시 인사이트로 자동 감지 — 확장 P1.)
  - **발행 실패 / 초안·컨펌만** → **미카운트**. 예약 발행은 **실제 발행된 날**에 칸 적립.
- 기준 타임존: **KST 자정** 기준 일자 귀속.

**2. 그리드·기간 (웹 우선)**
- **데스크톱 웹**: GitHub식 **연 단위 53주 가로 그리드**를 그대로 펼침(가로 공간 충분). 상단에 월 라벨, 좌측에 요일 라벨.
- **모바일 웹(반응형)**: 53주는 좁으므로 **최근 12~16주(약 3~4개월)로 축약 + 가로 스크롤**, [전체 보기]에서 연 단위.
- 구성: 가로 = 주, 세로 = 요일(월~일 7행). **오늘 칸 강조 테두리.**
- 색: 0 = 연회색, 1~4+ = 브랜드 컬러 농도 4단계. 하루 다발행은 **4+에서 캡**(과발행 압박 방지).

**3. 릴레이(streak) 표기 — "이어가기"가 본질**
- **현재 연속 발행 streak** — 일 단위 + **주 단위(주간 발행 지속률 연동)** 동시 표기. 예: "🔥 3주 연속 발행 중".
- **최장 기록(personal best)** — 갱신 시 축하 마이크로 인터랙션.
- **주간 목표 라인** — O4 진단의 주간 발행 목표(예: 주 2회)를 *주간 칸 합*과 비교해 "이번 주 1/2" 식 진척 배지.
- **끊긴 streak 복구 카피** — 죄책감 유발 금지. "이번 주 다시 시작해요" 같은 **재시작 유도**(이탈 방지). streak 0이어도 그래프는 과거 성취를 그대로 보존.

**4. 상호작용**
- **칸 hover** → 그날 발행한 콘텐츠 썸네일 + 핵심 지표 툴팁 → **클릭 시** 콘텐츠 성과(P1)/히스토리로. (모바일 웹은 탭→툴팁→재탭 이동.)
- **오늘 빈 칸** → [지금 만들기](K1) 진입 (행동 유도).
- **백필(선택)** — 서비스 가입 전·직접 올린 과거 게시물을 사용자가 수동 추가하도록 허용할지 → ⑨ Open Q.

|---|---|---|---|
| **업로드 릴레이(잔디)** | **행동** — 발행을 했는가 | 일/주 | 직접 통제 가능 (← 동기부여 핵심) |
| 100단위 로드맵 | **결과** — 팔로워 수 | 누적 | 간접 (따라오는 것) |
| 주간 루틴 트래커(H1) | 이번 주 할 일 | 주 | 직접 |

> **설계 의도**: 결과(팔로워)는 통제 밖이라 안 늘면 좌절 → 그래서 **통제 가능한 '발행 행동' 자체를 잔디로 보상**해 꾸준함을 지탱한다. 잔디는 "얼마나 컸나"가 아니라 **"얼마나 빠짐없이 했나"** 를 칭찬한다.

**6. 빈 상태 / 엣지 케이스**
- *온보딩 직후(잔디 0)*: 빈 그리드 + "첫 칸을 채워보세요" + [첫 콘텐츠 만들기](K1).
- 같은 날 여러 발행 → 농도 +1씩(4+ 캡). 발행 후 IG에서 삭제 → MVP는 잔디 유지(행동은 일어났음), 표기만 *주의: 삭제는 사람 확인 원칙이라 자동 회수 안 함*.
- IG 미연동 상태 → 내보내기 "올렸어요" 확인으로만 적립(자동 감지는 연동 후).

**7. 데이터 소스**
- 발행 이벤트(`status: published`, Graph API media id + 발행 timestamp) → 일자 집계.
- 내보내기 확인 이벤트(`exported` + user-confirmed) → 일자 집계.
- streak·주간 합은 위 집계의 파생값(별도 입력 없음).

### O3 · 톤앤매너 설정 (품질 근간)
- **선호 톤**(정보성/유머/감성/전문/친근) · **피할 스타일** · **브랜드 키워드 3~5** · **문체 예시** · **금지 문구** · **캡션 길이·해시태그·CTA 스타일** · **비주얼 가이드**.
- *결과*: 모든 생성 화면(K1~K3)에 제약으로 상속.

### O6 · 인스타그램 계정 연동
- **연동 안내** — 발행·DM·성과 자동화를 위해 필요함을 설명.
- **권한 범위 고지** — Graph API 권한(게시·인사이트·메시지).
- **연동 상태** — 연동 완료/실패/미연동.
- **[연동하기]** / **[나중에]**(연동 없이도 내보내기 경로로 사용 가능 — 단 IG 발행·DM·자동 성과는 비활성).

## ⑩ 인스타그램 인사이트 자동 수집 (성과 자동 채움)

> **연결**: O6/A4(IG 연동) 권한으로 수집 → P1(성과 기록) 자동 채움 → P2/P3 분석. **수동 입력은 보조 폴백.** (PRD v2 FR-14, P0)
> **기준 API**: Instagram Graph API v22.0. 스토리 지표는 **발행 후 24시간**만 조회 가능.

> **⏱️ 모든 수치에 공통 적용되는 시점·신선도 규칙 (공식 문서 근거)**
> 1. **응답 타임스탬프는 UTC(ISO-8601)** 로 내려온다 → 저장 시 **UTC 원본 + KST 변환** 두 값을 함께 적재(표시는 KST). [근거: Insights 개요 — *"Timestamps in API responses use UTC with zero offset … ISO-8601."*]
> 2. **최대 48시간 지연.** 계산 기반 지표는 확정까지 **최대 2일** 걸린다 → 오늘 새벽에 가져온 **어제·그제 값은 "잠정값"**, 며칠 뒤 재수집해 "확정값"으로 보정한다. [근거: User Insights 레퍼런스 — *"Data metrics … may have up to 48 hours of delay."*]
> 3. **계정 인사이트 보관 90일.** `follows_and_unfollows` 같은 시계열은 **최근 90일만** 조회 가능 → 그 이전은 우리 DB 적재본만 남는다. 즉 **매일 수집·적재가 필수**(놓치면 영구 결손). [근거: Insights 개요 — *"User Metrics data is stored for up to 90 days."*]

### 📌 "팔로워 100명" 기준에 대한 추가 설명

일부 인사이트는 **Meta 정책상 팔로워 100명 미만 계정에는 제공되지 않는다**(프라이버시 보호 목적). 우리 타깃이 **0~1,000명**이라 초반 사용자 상당수가 이 구간에 해당하므로, 설계 시 반드시 분기 처리한다.

- **100명 미만에도 되는 것 ✅**: 총 팔로워수(`followers_count`)와 모든 **게시물 단위 지표**(조회·도달·저장·공유 등). → **항목 1·2 및 4~11은 신생 계정에서도 정상 수집.**
- **100명 이상부터 되는 것 ⏳**: `follows_and_unfollows`(유입/이탈 분리, 항목 3), `follower_demographics`(인구통계). → 100명 미만에선 빈값/에러 반환.

> **⚠️ 이름 함정 (구현 주의)**: 인사이트 메트릭 **`follower_count`(단수)** = "하루에 늘어난 팔로워 수"는 **100명 제한 대상**이다. 우리가 총수로 쓰는 **`followers_count`(복수)** = "현재 총 팔로워수"는 **기본 필드라 제한 없음**. 단수/복수가 전혀 다른 것이므로 혼동 금지.
>
> **→ 설계 결론**: 하루 증감을 API가 직접 주는 `follower_count`(단수)는 100명에 막히므로, **100명 미만 구간에서는 총수 `followers_count`(복수)를 매일 찍어 차분으로 증감을 직접 산출**한다(항목 2). 사용자가 100명을 넘으면 대시보드가 자동으로 `follows_and_unfollows`(항목 3) **유입/이탈 분리 뷰로 업그레이드**된다.

> **근거(공식 문서)**: [Instagram User Insights API 레퍼런스](https://developers.facebook.com/docs/instagram-platform/api-reference/instagram-user/insights/) — *"`follower_count` and `online_followers` … not provided for … accounts with fewer than 100 followers"*, *"`follows_and_unfollows` … Not returned if the IG User has less than 100 followers"* · [Insights 개요](https://developers.facebook.com/docs/instagram-platform/insights/) — *"Some metrics are not available on Instagram accounts with fewer than 100 followers."* · [IG User 노드 — `followers_count` 필드](https://developers.facebook.com/docs/instagram-platform/instagram-graph-api/reference/ig-user/)
