# 테스트 3 — 인스타그램 연동·발행 검증 가이드 (비개발자용 따라하기)

> 목표: 내 테스트 인스타 계정에 **우리 코드로 카드뉴스를 실제 발행**해보고, 댓글→DM까지 동작하는지 확인.
> 핵심: **앱 리뷰는 필요 없습니다.** 개발 모드에서 내 계정으로 테스트하는 것이라 심사 전에 바로 가능.
> 진행 방식: 아래 단계를 **하나씩** 하시고, 각 단계가 끝나면 화면에 보이는 값(또는 막힌 지점)을 저(Claude)에게 알려주세요. 같이 진행합니다.

---

## 준비물 체크
- [ ] 인스타 계정 (보유 ✅)
- [ ] 페이스북 계정 — Meta 개발자 포털 로그인에 필요 (인스타 계정에 페이지 연결은 불필요, 포털 로그인용으로만 필요)
- [ ] 이메일

> ⚠️ 페이스북 계정이 없으면 Meta 개발자 포털에 못 들어갑니다. 없으면 하나 만들어야 해요(무료). 이 부분 알려주세요.

---

## STEP 1 — 인스타를 "프로페셔널 계정"으로 전환 (5분, 무료)
개인 계정은 API를 못 씁니다. 비즈니스/크리에이터로 바꿔야 해요. (언제든 되돌릴 수 있음)

1. 인스타 앱 → 프로필 → 우측 상단 메뉴(≡) → **설정 및 개인정보**
2. **계정 유형 및 도구** → **프로페셔널 계정으로 전환**
3. 카테고리 선택(아무거나, 예: "개인 블로그") → **비즈니스** 또는 **크리에이터** 선택
4. 완료

✅ 확인: 프로필에 "프로페셔널 대시보드"가 보이면 성공.
→ 끝나면 "1단계 완료"라고 알려주세요.

---

## STEP 2 — Meta 개발자 계정 + 앱 만들기 (10분)
1. 브라우저에서 **developers.facebook.com** 접속 → 페이스북 계정으로 로그인
2. 우상단 **시작하기 / 등록** → 개발자 등록(전화/이메일 인증)
3. **내 앱 → 앱 만들기**
4. 앱 유형: **비즈니스(Business)** 선택
5. 앱 이름(예: "1K Builder Test") 입력 → 생성

✅ 확인: 앱 대시보드 화면이 보이면 성공.
→ "2단계 완료"라고 알려주세요.

---

## STEP 3 — Instagram 제품 추가 (Instagram API with Instagram Login)
1. 앱 대시보드 → 좌측 **제품 추가** → **Instagram** 찾아서 **설정**
2. **"Instagram API with Instagram Login"** (= 인스타 비즈니스 로그인) 방식 선택
   - 이 방식은 페이스북 페이지 연결이 필요 없습니다.
3. 화면에 나오는 **Instagram 앱 ID / 앱 시크릿**, **OAuth 리디렉션 URI** 설정 항목을 확인
   - 리디렉션 URI는 테스트용으로 `https://localhost/` 같은 걸 임시로 넣어둡니다(이 부분 같이 정해요).

→ 여기서 화면에 보이는 항목들을 캡처하거나 알려주세요. **시크릿은 비밀**이니 일부만 가리고 알려주셔도 됩니다.

---

## STEP 4 — 내 인스타 계정을 테스터로 추가
앱이 개발 모드일 때, 역할이 있는 계정만 테스트할 수 있습니다.
1. 앱 대시보드 → **앱 역할 / 역할(Roles)** → **Instagram 테스터** 추가
2. 내 인스타 사용자명 입력 → 초대
3. 인스타 앱/웹 → 설정 → **앱 및 웹사이트 → 테스터 초대** → 수락

→ "4단계 완료"라고 알려주세요.

---

## STEP 5 — 액세스 토큰 받기 (가장 중요·같이 진행)
"토큰"은 우리 코드가 그 계정에 발행할 수 있게 해주는 열쇠입니다.
- 인스타 비즈니스 로그인 흐름으로 **인증 코드 → 단기 토큰 → 장기 토큰(60일)** 순으로 받습니다.
- 이 단계는 화면 흐름이 여러 개라, **여기서 멈추고 저와 같이** 한 단계씩 진행하는 게 안전합니다.
- 받은 토큰과 Instagram 사용자 ID는 **절대 공개 채널에 올리지 마세요.** 로컬 파일(.env)에만 둡니다.

---

## STEP 6 — 코드로 연결 확인 → 발행 테스트
토큰이 준비되면, 미리 만들어둔 스크립트 [poc/ig-test.js](poc/ig-test.js)로:
1. **연결 확인**: 내 계정 정보(사용자명, 팔로워 수)가 잘 불러와지는지
2. **단일 이미지 발행**: 공개 이미지 URL 1장을 실제 게시
3. **카드뉴스(캐러셀) 발행**: 우리가 만든 카드 PNG 여러 장을 한 게시물로

> 카드 PNG는 "공개 URL"이 있어야 인스타가 가져갑니다. 테스트용으로 임시 호스팅(예: 임시 이미지 호스팅, 또는 클라우드 스토리지)을 같이 정합니다.

---

## STEP 7 — 댓글 → DM 테스트 (선택, STEP 6 성공 후)
1. 발행된 게시물에 테스트 계정으로 키워드 댓글 달기
2. 웹훅으로 우리 서버가 이벤트 수신 → 자동 DM 발송 확인
- 이 단계는 "공개 HTTPS 주소"가 필요해서(ngrok 등) 별도로 같이 셋업합니다.

---

## 막히면?
각 단계에서 **화면 캡처 또는 보이는 텍스트**를 알려주시면 제가 다음 행동을 정확히 짚어드립니다. 에러 메시지도 그대로 주시면 됩니다.
