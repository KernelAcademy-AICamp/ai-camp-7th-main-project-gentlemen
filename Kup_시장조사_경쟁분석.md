## 0. 한눈 요약 (TL;DR)

- **Kup과 컨셉이 거의 동일한 직접 경쟁자는 사실상 `mirra` 하나.** 나머지는 파이프라인의 *일부*(카드뉴스 생성만 / DM 자동화만)만 겹친다.
- **mirra의 가장 공격적인 세일즈 포인트("완전 자동 Autopilot + AI 가상 페르소나")가 동시에 가장 큰 리스크였다** — Threads 스팸 판정·계정 정지(대표 본인 계정 포함) 논란 발생.
- 이 사건은 Kup의 기존 원칙(**휴먼인루프 검수 기본 ON · 공식 API만 사용 · 가짜 인게이지먼트 금지**)이 단순 보수성이 아니라 **시장에서 검증된 차별점**임을 보여준다.
- 가격대는 글로벌 기준 **월 $14~$40** 구간에 경쟁자가 밀집. Kup도 이 밴드를 의식한 가격 설계 필요.
- **Kup 고유 우위 3가지:** ① 안전한 자동화(정책 리스크 차단) ② "가상 인물"이 아닌 *실제 내 계정 성장* ③ 첫 1,000명(콜드스타트 통과)이라는 **명확한 목표·결과 지향** — 경쟁자 대부분은 "시간 절약"만 말하고 결과를 약속하지 않음.

## 1. 조사 범위 · 방법 · 한계

- **범위:** 국내(한국) + 글로벌의 "AI SNS 콘텐츠 생성·발행·DM 자동화" SaaS.
- **방법:** 공식 사이트 직접 확인(WebFetch) + 웹 검색(WebSearch) 교차 검증. mirra는 자사 페이지 + 제3자(Threads) 후기 양측 확인.
- **한계/주의:**
  - 가격·기능은 **조사 시점(2026-06) 기준**이며 변동 가능. 특히 ManyChat은 2026-03 가격 개편이 있었음.
  - mirra의 `/autopilot`, `/persona-technology` 페이지는 **직접 fetch 시 404**(Vercel 클라이언트 렌더링/봇 차단 추정)이나, **Vercel 미러(`mirraa.vercel.app`)와 검색 스니펫으로 내용 확보 완료**(§2.5 참조).
  - "AI 페르소나 사태"는 제3자(Threads `@andytechcan`) 후기 기반 정황이며, mirra 공식 입장은 별도 확인 필요.

## 2. 직접 경쟁자 심층 분석 — mirra (미라)

> https://www.mirra.my/ko · "기획부터 발행·관리까지" AI 마케팅 플랫폼

### 2.1 무엇인가
한 번 입력하면 글·디자인·영상까지 멀티포맷으로 생성하고, 다채널 예약 발행, 댓글·DM 응대까지 하는 **"AI 전담 마케팅 팀"**을 표방. **컨셉상 Kup과 가장 직접적으로 겹치는 서비스.**

### 2.2 기능·가격 정리

| 항목 | 내용 |
|---|---|
| 콘텐츠 생성 | 카드뉴스(캐러셀, ~5분), SNS 포스트(6개 플랫폼 최적화), 블로그(2,000자+ SEO), 숏폼(~3분) |
| 지원 채널 | Threads, Instagram, X(트위터), YouTube, **TikTok**, LinkedIn (재조사 시 TikTok 확인) |
| 발행/관리 | 통합 캘린더 예약발행 + 최적 발행시간 추천 + 댓글·DM 자동 응대/우선순위 분류 |
| 분석 | AI 리포트 + 인사이트 챗봇, ROI 추적 |
| AI 차별 | 계정 기존 글 학습 → **말투·톤 자동 복제**("브랜드/페르소나 학습") |
| 승인 | "미승인 콘텐츠는 게시되지 않음 / 발행 전 사용자 승인" 표방 (단, Autopilot 라인은 완전 자동 지향) |
| 가격 | **월 $19부터**, 7일 무료체험. "월 $100+·매일 4시간" vs "월 $19·10분" 식 비교 마케팅 |
| 성과 소구 | "마케팅 시간 90% 절감", "3개월 3천 팔로워", "월 400만원 매출" 사례 |

### 2.3 ⚠️ 핵심 리스크 — "AI 페르소나 사태"
mirra는 **여러 가상 인물(페르소나) 계정을 만들고 AI가 말투까지 흉내내 완전 자동으로 글을 올리는** 방식을 핵심으로 밀었고, 이 지점에서 논란이 발생했다 (출처: Threads `@andytechcan` 정리글):

1. **윤리/투명성** — "실존하지 않는 가상 인물을 AI가 운영하는 게 도덕적으로 맞나"
2. **플랫폼 정책 위반** — Threads가 이런 자동화 계정을 **스팸으로 처리**하는 정황
3. **계정 대량 삭제** — 운영하던 자동화 계정 삭제, **결국 대표(박규태) 본인 계정까지 정지**

> **시사점:** "완전 자동 + 가상 인물"의 부작용(계정 정지)이 시장에서 이미 표면화됨. Kup의 *공식 API only · 휴먼인루프 · 실계정 성장* 노선이 안전성 측면에서 유리.

### 2.4 Autopilot · 페르소나 기술 (404 페이지 재조사 결과)

> `/autopilot`·`/persona-technology`는 직접 fetch가 404였으나, Vercel 미러·검색으로 내용 확보.

**Autopilot(완전 자동 발행)**
- 카피: *"AI가 브랜드를 학습해 매일 콘텐츠를 기획·작성·발행. **당신은 아무것도 안 해도 됩니다.**"*
- = 주제 선정→초안→발행까지 사람 개입 없이 도는 **완전 자동 라인**.

**페르소나 기술 — 4가지 방식**
1. **자동 학습** — SNS 계정 연동 시 기존 글에서 말투·톤·주제 취향 자동 파악
2. **맞춤 페르소나 설계** — AI와 대화로 브랜드 컨셉·타깃·말투를 *처음부터* 정의 (실존 인물과 무관한 가상 페르소나 생성 가능)
3. **템플릿** — 마케터·크리에이터·전문가 등 사전 검증된 페르소나 템플릿 즉시 적용
4. **승인 프로세스** — 트렌드 분석→주제 추천→초안→(편집/승인)→최적 시간 자동 발행

### 2.5 ⚠️ 메시지 모순 — "승인 필수" vs "아무것도 안 해도 됨"

mirra 마케팅 페이지는 *"승인하지 않은 콘텐츠는 절대 발행되지 않습니다"*(휴먼인루프)를 내세우지만, **Autopilot 라인은 *"당신은 아무것도 안 해도 됩니다"*(완전 자동)**를 약속한다. 제3자 후기(Threads)는 실제로는 **가상 페르소나 다계정을 AI가 완전 자동 운영**하는 시스템으로 묘사 → 결국 계정 정지로 이어짐.

> **해석:** mirra는 "안전(승인)"과 "완전 자동(편함)"을 동시에 약속하지만, 실사용에서 후자가 부각되며 정책 리스크가 터졌다. **Kup은 둘을 동시에 약속하지 말고 "안전(검수 기본 ON)"을 단일 메시지로 못 박는 것이 차별·신뢰에 유리.**

### 2.6 mirra vs Kup 정면 비교

| 축 | mirra | Kup |
|---|---|---|
| 컨셉 | AI 완전 자동 + 가상 페르소나 | AI 생성 + **휴먼인루프 검수 기본 ON** |
| 발행 | Autopilot(완전 자동 지향) | 발행 전 사람 승인 / 삭제는 자동화 안 함 |
| 계정 전략 | 가상 인물 다계정 | **실제 운영자 1계정 성장** |
| API/정책 | 자동화 적극 → 스팸 판정·정지 사례 | **공식 Graph API만 / 자동팔로우·콜드DM 금지** |
| 목표 약속 | "시간 90% 절감"(결과는 비보장) | **첫 1,000명(수익화 진입선) 도달** = 결과 지향 |
| 다국어 | 한국어 중심 | 영·일·중 카드 렌더 검증(현재는 한국 우선·다국어 Out) |
| 타깃 | 사업가·마케터·전문직·대행사 | 인스타 **입문** 예비창작자(콜드스타트) |

## 6. Kup 차별화 · 포지셔닝 제안

1. **"안전한 자동화"** — mirra의 계정정지 사태를 반면교사로, *공식 API + 사람 검수 = 정책 안전성*을 전면에. (이미 PRD §5의 안전장치와 일치 → 마케팅 메시지로 승격)
2. **"가짜 인물이 아니라 진짜 내 계정"** — 가상 페르소나 다계정이 아닌, *실제 운영자의 브랜드·계정을 키운다*는 신뢰 포지션.
3. **"시간 절약이 아니라 결과"** — 경쟁자는 "90% 시간 절감"을 말하지만 Kup은 **"콜드스타트(0→1,000명) 통과"라는 결과**를 약속. 차별화의 핵심.
4. **세그먼트 차별** — 경쟁자 대부분은 "이미 파는 사람(셀러·전문직·대행사)" 타깃. Kup은 **"이제 막 시작하는 입문자"**라는 미충족 시장(가장 어렵고·이탈 많은 구간)을 정조준.
5. **가격 설계** — 글로벌 표준 $14~$40 밴드 인식. **연락처 기반 과금(ManyChat형)은 회피** — 팔로워를 늘려주는 서비스가 연락처로 과금하면 메시지 충돌.
