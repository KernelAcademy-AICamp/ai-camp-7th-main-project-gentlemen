# Kup IA — 설명(M열) 초안

> **용도**: 팀 IA 스프레드시트(`KUP IA`)의 **설명 열**에 바로 붙여넣을 초안. 행 번호(No)는 스프레드시트와 1:1.
> **근거**: 최종 PRD v1.0 + 우리가 정리한 화면 스펙(업로드 릴레이·검수 게이트·연동 게이트·다음 액션). 화면 ID(K/P/G/A…)는 `Kup_IA_초안.md` 대조용.
> **표기**: `[제안]` = 스프레드시트에 아직 없지만 PRD상 필요해서 내가 더한 노드. `[보강]` = 있는 노드에 PRD 핵심을 덧댄 부분.

## 정합성 메모 (스프레드시트 채울 때 결정할 4가지)

1. **전략 진단 설문(온보딩)** — 행 21~62엔 안 보임. **0~0.1 영역에 있는지 확인.** 없으면 PRD 차별점 1이 비므로 0.4.4(가입완료) 뒤 워크스페이스 first-run으로 추가.
2. **검수 하드게이트** — 칸반(0.2.3.4)에 검수 컬럼이 없음. **제작완료 → 예약/업로드 전환 시 검수(0.2.3.3.2) 강제**임을 설명에 명시(위 초안 반영).
3. **다음 액션 추천** — 인사이트(0.2.3.5)를 데이터 표기로 끝내지 말고 "다음에 바꿀 점" 코칭 추가(위 [보강] 반영) → PRD 운영지속성 루프 완성.
4. **DM 리드마그넷 설정 화면** — 인사이트엔 카운트만 있음. **설정 동선(0.2.x [제안])** 위치 결정 필요.
