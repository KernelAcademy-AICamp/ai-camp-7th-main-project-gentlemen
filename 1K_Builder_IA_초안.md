## ① IA 설계 원칙 (PRD 제품 원칙에서 도출)

| 원칙 | IA에 주는 규칙 |
|---|---|
| **주체성 보존** | 모든 콘텐츠 화면의 종착점은 "발행"이 아니라 **"내보내기(사용자가 직접 올림)"**. AI 산출물은 항상 *수정 가능한 초안* 상태로 노출. |
| **검수 = 직접 수정 + 최종 컨펌** | 초안 → **에디터** → **최종 컨펌**을 끊기지 않는 한 흐름으로. 컨펌 화면 없이 내보내기로 못 건너뜀(하드 게이트). |
| **시간 단축이 가치** | 홈은 '설정 모음'이 아니라 **"이번 주 할 일 + 진행 중 콘텐츠"** 허브. 제작까지 최소 탭. |
| **성과→다음 액션 루프** | 성과는 숫자 나열이 아니라 **"다음에 바꿀 점"**으로 끝나고, 그게 다시 제작(주제 추천)으로 연결되는 순환 구조. |
| **작은 목표** | 1,000을 단일 게이지로 두지 않고 **100/500/1,000 마일스톤 트래커**를 홈에 상시 노출. |
| **점진적 노출(세그먼트)** | 초보·소상공인 공통 코어 + 소상공인 전용 비즈니스 모듈은 **온보딩 목적에 따라 토글**. 초보에게 불필요한 화면을 숨김. |

## ② 전체 구조 (사이트맵)

```
Kup
│
├─ [최초 1회] 온보딩
│   ├─ O1 계정 목적·주제 입력          ─┐ 세그먼트 분기점
│   ├─ O2 계정 상태 입력 (팔로워·기간·업로드 빈도·형식)
│   ├─ O3 톤앤매너 설정
│   ├─ O4 진단 결과 (운영 단계 + 이번 달/주 목표)
│   └─ O5 계정 운영 체크리스트
│
├─ [탭1] 홈 · 대시보드  ★허브
│   └─ H1 오늘의 코치 (이번 주 목표·진행 중 콘텐츠·성과 입력 대기·마일스톤·다음 액션)
│
├─ [탭2] 콘텐츠 (제작 허브)
│   ├─ K1 주제 추천 (이번 주 5개)
│   ├─ K2 초안 생성 결과 (제작 패키지)
│   ├─ K3 직접 수정 에디터
│   ├─ K4 최종 컨펌 & 내보내기
│   └─ K5 비즈니스 문구 도구  ◆소상공인(C)
│
├─ [탭3] 성과
│   ├─ P1 성과 기록 입력
│   ├─ P2 다음 액션 추천
│   └─ P3 성과 히스토리 · 마일스톤
│
├─ [탭4] 내 계정
│   ├─ A1 계정 홈 (현재 단계·프로필 요약)
│   ├─ A2 톤앤매너 프로필 (편집)
│   ├─ A3 운영 체크리스트 (재방문)
│   └─ A4 설정 (구독·알림·계정)
│
└─ [확장 자리] (P1/P2 — 아래 ⑧)
    ├─ R  레퍼런스 아카이브            (P1)
    ├─ CAL 시즌/이벤트 캘린더          (P1)
    ├─ TR  트렌드 오디오·해시태그       (P1)
    └─ IG  인스타 연동 (읽기 P1 / 쓰기·자동발행·코멘트→DM P2)
```

★ = 기본 진입 허브 · ◆ = 소상공인 세그먼트에서만 노출

## ③ 내비게이션 모델

**구조: 하단 탭 4개 + 홈 허브(hub-and-spoke).** (모바일 우선 — 사용자는 인스타를 폰에서 운영)

```
┌─────────────────────────────────┐
│            (현재 화면)            │
│                                 │
├─────────────────────────────────┤
│  🏠 홈   ✍️ 콘텐츠   📊 성과   👤 내 계정 │
└─────────────────────────────────┘
```

- **홈(탭1)이 모든 행동의 시작점.** "이번 주 목표 → 추천 주제 → 진행 중 콘텐츠 → 성과 입력 대기"를 카드로 띄워 사용자가 *다음에 뭘 할지 고민하지 않게* 한다.
- **콘텐츠(탭2)는 제작 파이프라인.** K1→K2→K3→K4가 한 방향 흐름. 중간 이탈 시 초안은 '진행 중'으로 저장되어 홈에 다시 뜬다.
- **성과(탭3)는 루프의 후반.** 내보낸 콘텐츠가 여기서 '성과 입력 대기'로 쌓이고, 입력하면 다음 액션 → 다시 콘텐츠 제작으로 연결.
- **내 계정(탭4)은 무거운 설정.** 톤앤매너·체크리스트·구독. 자주 안 가지만 톤 프로필은 품질의 근간.

**근거:** PRD의 핵심 루프(진단→추천→초안→수정→내보내기→성과→다음액션)는 *순환*이다. 평면적 메뉴보다 **홈 허브가 매 순간 "지금 할 한 가지"를 제시**하는 구조가 '꾸준히 못 함' 문제(Pain #1)에 직접 대응한다.

> **대안 검토:** 탭 없이 단일 위저드(온보딩처럼 쭉 진행)도 가능하나, 반복 사용(주 2~3회 제작 + 성과 입력)에는 탭형 재진입이 유리. → 탭형 채택.

## ⑤ 핵심 화면별 정보구조 (P0)

> 표기: **블록**(우선순위 순) · *상태*(빈/생성중/완료) · [액션]

### H1 · 오늘의 코치 (홈 허브)
- **이번 주 목표 배너** — O4 진단의 이번 주 콘텐츠 목표(예: "릴스 2개 + 카드뉴스 1개"). 진척 표시.
- **다음 할 한 가지(Primary)** — 상태에 따라 1개만 강조: 진행 중 초안 있으면 [이어서 수정], 없으면 [이번 주 주제 받기], 성과 미입력 콘텐츠 있으면 [성과 입력].
- **진행 중 콘텐츠** — draft/edited 상태 카드 리스트 → K3로.
- **성과 입력 대기** — exported인데 measured 아닌 콘텐츠 → P1으로.
- **마일스톤 트래커** — 현재 팔로워 / 다음 목표(100→500→1K). 작은 성취 강조.
- **다음 액션 요약** — 최근 P2 추천 1~2개 재노출.
- *빈 상태(온보딩 직후)*: 목표 배너 + [첫 콘텐츠 만들기] 단일 CTA만.

### K1 · 주제 추천
- **이번 주 추천 5개(카드)** — 각 카드: 주제 한 줄 · **목적 태그**(조회/저장/공유/방문/문의) · **추천 포맷**(릴스/카드뉴스/스토리) · 후킹 방향 · 필요 촬영 소스.
- **[다시 추천]** / **[직접 주제 입력]** 보조 액션.
- 소상공인◆: 추천에 **시즌/이벤트 앵글** 우선 노출(예: "가정의 달 클래스").
- *생성 중*: 스켈레톤 카드. → [이 주제로 초안] 누르면 K2.

### K2 · 초안 생성 결과 (제작 패키지)
- **포맷 헤더** — 선택 포맷·목적 재확인.
- **콘텐츠 본문 블록**(포맷별):
  - 릴스 → 후킹 문구 · 30/60초 대본 · 자막 초안 · (얼굴 등장/첫 3초 가이드)
  - 카드뉴스 → 페이지별 구성(표지~마무리) · 썸네일 문구
- **메타 블록** — 캡션 · 해시태그 · CTA.
- **[수정하기](Primary)** — K3로. (K2는 '읽기', K3는 '편집' — 분리해 한눈에 검토 후 편집)
- *상태*: 생성 중(부분 스트리밍 표시) → 완료.

### K3 · 직접 수정 에디터 (차별화 핵심)
- **블록 단위 편집** — 각 블록(후킹·대본·캡션·해시태그…) 인라인 텍스트 직접 수정.
- **블록별 보조 도구**(블록 선택 시 노출):
  - **톤 변경** 버튼 — 더 친근/정보성/짧게/감성적으로 → 재생성
  - **문장 대체안** — 2~3개 변형 중 선택
  - **해시태그** — 추천 목록에서 선택/삭제
- **AI 자가 점검 플래그** — 미검증 주장·실존 인물·효능·이미지 아티팩트 등 *의심 지점만* 상단 배지로. (빈 검수 방지)
- 소상공인◆: **금지어/브랜드 톤 위반 경고** 인라인 표시(톤 프로필 기반).
- **[최종 확인으로](Primary)**.
- *상태*: 미수정/수정됨 표시 → status: edited.

### K4 · 최종 컨펌 & 내보내기 (하드 게이트)
- **발행 전 체크리스트** — 톤 확인·고위험 플래그 해소·해시태그·CTA 점검(체크 안 하면 내보내기 비활성).
- **최종 미리보기** — 인스타 게시 형태(캡션+이미지/카드) 미리보기.
- **내보내기(Primary)** — [캡션 복사] · [카드뉴스 이미지 내보내기] · [대본/자막 텍스트 내보내기]. *MVP는 자동 발행 없음 — 사용자가 직접 올림.*
- **[성과 입력 예약]** — 내보내면 자동으로 '성과 입력 대기'에 등록(H1·P1로 환류).
- *완료*: status: exported.

### P1 · 성과 기록 입력
- **대상 콘텐츠 선택** — exported 목록에서.
- **지표 입력(수동)** — 조회/도달·좋아요·댓글·저장·공유·평균 시청시간·프로필 방문·팔로우 증가·링크 클릭/DM 문의. (포맷별 관련 지표만 노출해 입력 부담↓)
- **[저장 후 분석](Primary)** → P2.
- *주의(PRD 리스크)*: 수동 입력 부담이 핵심 이탈 위험 → **꼭 필요한 지표만, 단계적 입력**. (P1 단계 IG 읽기 연동 시 자동 채움)

### P2 · 다음 액션 추천
- **이번 콘텐츠 강점** — 잘 나온 지표.
- **부족했던 지표**.
- **다음에 바꿀 점** — 규칙 기반 2~3개(조회↔전환·시청시간·저장·공유·방문↔문의 매핑).
- **[이 실험으로 주제 받기](Primary)** → K1에 컨텍스트 전달(루프 완성).

### O3 · 톤앤매너 설정 (품질 근간)
- **선호 톤 선택**(정보성/유머/감성/전문/친근) · **피하고 싶은 스타일**(어그로/기계적/과한 이모지/조잡 이미지).
- **브랜드 키워드 3~5** · **문체 예시**(직접 입력 or 게시물 붙여넣기) · **금지 문구**.
- **캡션 길이·해시태그·CTA 스타일** · **비주얼 가이드**(색감·무드·스톡 사용 여부).
- *결과*: 모든 생성 화면(K1~K5)에 제약으로 상속.

> 데이터 모델(PRD §13)이 어디서 생성·소비되는지. status 흐름이 IA 동선과 일치.

| 객체 | 생성 화면 | 소비/갱신 화면 | status 흐름 |
|---|---|---|---|
| **계정 프로필** (niche·goal·followers·capacity 등) | O1·O2 | A1, H1(목표), K1(추천 입력) | — |
| **톤앤매너** (tone_keywords·banned·visual) | O3 | A2(편집), K1~K5(상속) | — |
| **콘텐츠** (hook·script·carousel·caption·hashtags·CTA) | K2(생성) | K3(edited)→K4(exported)→P1(measured) | draft→edited→exported→measured |
| **성과** (reach·saves·shares·watch_time·next_action) | P1(입력) | P2(분석), P3(히스토리), H1(요약) | — |
| **마일스톤** (current_followers vs 100/500/1K) | 파생 | H1·P3 | — |

## ⑦ 세그먼트 분기 (초보 A ↔ 소상공인 C)

**분기점 = O1 운영 목적.** 취미/브랜딩/포트폴리오 → 초보 코어. 매출/클래스 문의/예약 → 소상공인 모듈 ON.

| 영역 | 초보(A) | 소상공인(C) 추가 |
|---|---|---|
| 온보딩 | 기본 | 비즈니스 정보(업종·매장·예약 링크) 1스텝 추가 |
| 주제 추천(K1) | 조회·저장·공유 목적 위주 | **문의·방문·예약 유도** 목적 + 시즌/이벤트 앵글 |
| 제작(K2/K3) | 공통 | **제품/클래스 소개 톤** 프리셋 |
| 비즈니스 도구(K5) | 숨김 | **DM/문의 응대·예약 CTA 문구**(P0 일부) |
| 성과(P2) | 조회·팔로우 중심 | **프로필 방문→문의/예약 전환** 액션 우선 |
| 마일스톤 | 팔로워 100/500/1K | + 문의/예약 유입(보조 지표) |

> 코어 파이프라인(K1→K4·P1→P2)은 **동일**. 소상공인은 *추가 모듈 토글*이지 별도 앱이 아니다 → 유지보수·학습 비용 최소화.

## ⑧ 확장 자리 (P1 / P2) — IA에서 미리 비워둠

| 기능 | 들어올 위치 | IA 영향 |
|---|---|---|
| **레퍼런스 아카이브**(P1) | 탭 추가 또는 콘텐츠 탭 내 섹션 | "저장 → 내 계정 맞게 변환 → K2 초안"으로 K1 대체 진입 경로 신설 |
| **시즌/이벤트 캘린더**(P1) | 홈 카드 + 콘텐츠 탭 | K1 추천에 캘린더 기반 주제 주입 |
| **트렌드 오디오·해시태그**(P1) | K3 에디터 내 보조 패널 | 정책 검토 후 |
| **IG 읽기 연동**(P1) | A4 설정 → 연동 | P1 성과 자동 수집(수동 입력 대체) |
| **IG 쓰기·자동 발행·코멘트→DM**(P2) | K4에 [예약 발행] 추가 · 신규 'DM 리드마그넷' 화면 | 자동 발행은 **옵트인 토글 + 킬스위치**로. 베타는 테스터 한정(개발 모드) |

> 자동 발행이 들어와도 **K4 최종 컨펌 게이트는 유지** — '승인 후 발행'이지 무검수 발행이 아니다(PRD 원칙).

## ⑨ Open Questions (IA 차원)

- [ ] **홈 허브 vs 위저드** — 탭형 채택했으나, 초보에게 첫 2주는 위저드형 가이드가 나을 수도. A/B 검토?

- [ ] **K5 비즈니스 도구를 별도 탭 vs 콘텐츠 탭 내** — 소상공인 비중이 크면 별도 탭 승격 검토.
- [ ] **성과 수동 입력 UX** — 전체 입력 vs "핵심 3개만 빠른 입력". 이탈 방지가 관건(PRD 최대 리스크).
- [ ] **마일스톤 표시 위치** — 홈 상시 vs 성과 탭. 동기부여 효과 측정 후.
